# memory/ — Система памяти

> Хранит контекст между сессиями. Читается агентами при старте.

---

## Структура

```
memory/
├── active-context.md      ← ОБЯЗАТЕЛЬНО читать при старте
├── README.md              ← этот файл
└── YYYY-MM-DD.md          ← дневные логи (создаются автоматически)
```

---

## Правила

1. **active-context.md** — обновляется после КАЖДОЙ задачи
2. Дневные логи — создаются при значимых событиях
3. Файлы старше 30 дней — архивируются или удаляются
4. Максимум 15-20 активных записей в total

---

## Формат дневного лога (YYYY-MM-DD.md)

```markdown
# Лог YYYY-MM-DD

## Что сделано
- (задача 1)
- (задача 2)

## Решения и договорённости
- (важное решение)

## Проблемы и решения
- Проблема: (описание) → Решение: (описание)

## Следующий шаг
- (что делать в следующую сессию)
```
